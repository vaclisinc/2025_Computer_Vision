Part 1.1: Convolutions from Scratch

I implemented 2D convolution three ways to understand performance trade-offs:

Three Convolution Implementations

▶ Implementation 1: Naive (4 loops) 

def convolve_naive(image, kernel):
    h, w = image.shape
    kh, kw = kernel.shape

    # Padding
    pad_h = kh // 2
    pad_w = kw // 2
    padded = np.pad(image,
        ((pad_h, pad_h), (pad_w, pad_w)),
        mode='constant')
    output = np.zeros((h, w))

    for i in range(h):
        for j in range(w):
            for ki in range(kh):
                for kj in range(kw):
                    # Flip kernel
                    fi = kh - 1 - ki
                    fj = kw - 1 - kj
                    output[i,j] += padded[i+ki, j+kj] * kernel[fi, fj]
    return output

▶ Implementation 2: Optimized (2 loops) 

def convolve_optimized(image, kernel):
    h, w = image.shape
    kh, kw = kernel.shape

    # Padding
    pad_h = kh // 2
    pad_w = kw // 2
    padded = np.pad(image,
        ((pad_h, pad_h), (pad_w, pad_w)),
        mode='constant')

    output = np.zeros((h, w))

    # Pre-flip kernel
    flipped = np.flip(np.flip(kernel, 0), 1)

    for i in range(h):
        for j in range(w):
            region = padded[i:i+kh, j:j+kw]
            output[i,j] = np.sum(region * flipped)

    return output

▶ Implementation 3: Using SciPy 

def convolve_scipy(image, kernel):
    output = scipy.signal.convolve2d(
        image, kernel,
        mode='same',
        boundary='fill',
        fillvalue=0)
    return output

Testing with 9×9 Box Filter

To test our implementations, I used a 9×9 box filter created with:

box_filter = np.ones((9, 9)) / (9 * 9)

This creates a uniform averaging filter where each pixel is replaced by the mean of its 9×9 neighborhood.

Finite Difference Operators

Next, I tested with finite difference operators to detect edges:

Dx = np.array([[-1, 0, 1]]) # Detects vertical edges Dy = np.array([[-1], [0], [1]]) # Detects horizontal edges

These operators compute the image gradient by approximating derivatives in x and y directions.

Implementation Dx Runtime Dy Runtime
Naive 1.47s 1.60s
Optimized 2.89s 2.92s
Scipy 0.01s 0.02s

Key Observations

• Small kernels (3×3) show naive faster than optimized due to NumPy overhead.

• All implementations produce identical results.

• Dx, Dy outputs: Can be negative or positive, so black = negative, gray = zero, white = positive.

• Gradient Magnitude: Always positive, so black = 0 (no edge), white = strong edge.
